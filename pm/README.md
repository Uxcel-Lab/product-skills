# PM Skills

21 skills for product-management work — from framing a problem to building specs, roadmaps, and metrics, and reviewing them. Each links to its `SKILL.md`.

**Kinds:** _foundations / deliverables / processes_ are **generative** (they gate decisions on context); _audits_ are **evaluative** (they review work and return severity-rated fixes); _workflows_ are **orchestrators** (they compose the others).

---

## Foundations

The framing artifacts everything else builds on.

| Skill | Purpose |
|---|---|
| [problem-statement](./foundations/problem-statement/SKILL.md) | Turn a vague prompt or request into a clear, evidence-grounded, testable statement of the real problem. |
| [personas-jtbd](./foundations/personas-jtbd/SKILL.md) | Research-based personas or a Jobs-to-Be-Done framing — and the choice between them; never invented fiction. |
| [user-story](./foundations/user-story/SKILL.md) | Write user stories framed around who/what/why, kept INVEST and testable; or critique existing ones. |

## Deliverables

The documents PMs ship.

| Skill | Purpose |
|---|---|
| [product-spec](./deliverables/product-spec/SKILL.md) | Write a PRD/spec anchored to a validated problem, with measurable requirements and surfaced risks. |
| [roadmap](./deliverables/roadmap/SKILL.md) | Outcome-oriented, theme-based, confidence-aware (Now-Next-Later) roadmaps tied to strategy. |
| [okrs-kpis](./deliverables/okrs-kpis/SKILL.md) | OKRs, KPIs, or a North Star Metric with outcome-based key results, baselines, targets, and guardrails. |
| [vision-strategy](./deliverables/vision-strategy/SKILL.md) | An aspirational destination plus the trade-off-driven choices to reach it — not a buzzword poster. |
| [competitive-analysis](./deliverables/competitive-analysis/SKILL.md) | A competitive/market analysis that yields a strategic edge, not a flat feature table. |
| [gtm-plan](./deliverables/gtm-plan/SKILL.md) | A stage-appropriate go-to-market system — customers, channels, pricing, timing, partnerships. |

## Processes

How PMs work — discovery, prioritization, testing, analytics.

| Skill | Purpose |
|---|---|
| [discovery](./processes/discovery/SKILL.md) | Plan continuous discovery — figure out what to build before building it; match evidence to risk. |
| [prioritization](./processes/prioritization/SKILL.md) | Decide what to build first — right framework, scoring that exposes trade-offs, sequencing, saying no. |
| [assumption-testing](./processes/assumption-testing/SKILL.md) | Decompose an idea into load-bearing beliefs, test the riskiest with the cheapest method. |
| [experimentation-ab](./processes/experimentation-ab/SKILL.md) | Design A/B tests with rigor — pre-registered hypothesis, power up front, guardrails, a decision rule. |
| [analytics](./processes/analytics/SKILL.md) | Analysis that produces a decision, not a dashboard — actionable metrics, segmentation, insight→action. |

## Audits _(evaluative)_

Review existing PM work against a standard and return a prioritized, severity-rated issue list with fixes.

| Skill | Purpose |
|---|---|
| [spec-quality](./audits/spec-quality/SKILL.md) | Audit a spec/PRD — untestable requirements, happy-path-only flows, missing scope and assumptions. |
| [prioritization-rigor](./audits/prioritization-rigor/SKILL.md) | Audit scored backlogs and roadmap orderings — false precision, strategy drift, must-have inflation. |
| [okr-metric-validity](./audits/okr-metric-validity/SKILL.md) | Audit OKRs/KPIs/metrics — vanity metrics, unfalsifiable KRs, outputs disguised as outcomes, gameable targets. |
| [assumption-rigor](./audits/assumption-rigor/SKILL.md) | Audit how a team validated its assumptions — riskiest-vs-comfortable, method fidelity, after-the-fact criteria. |
| [decision-quality](./audits/decision-quality/SKILL.md) | Audit a decision or its rationale — resulting, reversibility mismatch, bias, missing pre-mortems. |

## Workflows _(orchestrators)_

| Skill | Purpose |
|---|---|
| [product-review](./workflows/product-review/SKILL.md) | Review a PM artifact — routes by type to the right audits, runs the assumption check, returns one report. |
| [idea-to-plan](./workflows/idea-to-plan/SKILL.md) | Idea to build-ready plan — composes problem → discovery → users → test → prioritize → spec → metrics. |
